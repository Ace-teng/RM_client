# 客户端交接说明文档（给下一位同学）

> 目标：你看完这份文档后，应当对**整个项目结构、数据流、UI 架构和注意事项**有比较高的理解度，能安全地继续开发。

---

## 1. 项目总体概览

- **项目目标**：为 RoboMaster 比赛制作一套**自定义客户端**，供操作手观战、瞄准、查看状态和做战术决策。
- **技术栈**：
  - 比赛用客户端：**Python + QtPy (PyQt/PySide)**。
  - 预览用前端：**React + TypeScript**（只做 UI 预览，不参与比赛）。
- **关键目录**：
  - `rm_client/`：**实际比赛用的 Python 客户端**（核心）。
  - `docs/`：项目文档（架构说明、功能说明、交接说明）。
  - 其它如 `src/` / `web` / `preview` 等：**React 预览项目，用于 UI 设计参考，不直接参与比赛。**

> 若只关心比赛客户端，优先看：`rm_client/` + `docs/`。

## 3. 顶层架构：Python 客户端

### 3.1 Core 层（数据与业务逻辑）

- **`rm_client/core/model/datacenter.py`**
  - 全局单例 **`DataCenter`**，是 UI 与业务的**唯一数据桥梁**。
  - 关键字段：
    - `operator_display_state`：HUD 所需对象（己方/敌方 HP、能量、电容、自瞄状态、比赛时间、热量等）。
    - `game_state`：比赛阶段、剩余时间。
    - `robot_states`：小地图用到的机器人坐标、阵营信息。
    - `link_status`：MQTT 等连接状态。
    - `tactical_advice`：战术建议（例如“对方可能买活”）。

- **`rm_client/core/service/*`**
  - 例如 `tactical_advisor.py`：
    - 控制 **买活预警** 的触发条件。
    - 逻辑已经改过：不会在刚启动/占位数据时就弹预警。

> 改动指引：  
> - 想改 **HUD 显示的数据逻辑** → 看 `DataCenter` 及填充它的各个 `service`。  
> - 想改 **UI 布局或样式** → 看 `rm_client/ui/` 下面的文件。

### 3.2 UI 总框架

- **`rm_client/ui/main_window.py`**（主窗口）
  - 使用 `QMainWindow`，中心内容为一个垂直布局：
    - 顶部：`InfantryTopBar`（步兵专用顶栏）。
    - 中部：`QSplitter(Qt.Horizontal)` 分成三列：
      1. **左列**：`InfantryLeftPanel`（步兵专用左侧竖栏）。
      2. **中列**：`VideoArea`（图传 + HUD + 小地图）。
      3. **右列**：`right_wrapper`，包含：
         - `OperatorRoleSelector`：角色选择下拉框（英雄 / 步兵1 / 步兵2）。
         - `_right_stacked`：英雄视图 vs 步兵视图的右侧面板。
    - 底部：`statusBar` 显示整体状态。

  - 右侧堆叠内容：
    - **英雄页**：
      - `RightInfoPanel`：连接、阶段、剩余时间、血量、复制诊断。
      - `PLUGINS` 区域：可挂动态插件控件。
    - **步兵页**：
      - `InfantryRightPanel`：四块区域：
        - SYSTEM STATUS / SCORES / MODULE HEALTH / CHAT LOG。

  - **角色切换逻辑**：
    - `OperatorRoleSelector` 改变 `DataCenter.current_operator_role`。
    - `_sync_layout_to_role()` 周期性读取角色，控制：
      - 顶栏显隐（仅步兵可见）。
      - 左侧面板显隐（仅步兵可见）。
      - 右侧堆叠页的当前索引（英雄 or 步兵）。
      - 底部状态栏文本（步兵时显示 SYSTEM ACTIVE / PING / SIGNAL 等）。

- **`rm_client/ui/video_area.py`**（图传 + HUD + 小地图）
  - 内部组件：
    - `_graphics_view` + `_scene` + `_pixmap_item`：显示视频帧。
    - `_placeholder_text`：无信号时的 **“NO SIGNAL”** 文本。
    - `_overlay: BuybackOverlay`：屏幕中央的买活提示。
    - `_hud_overlay: VideoHudOverlay`：画在图传上的 HUD 层（血量、时间、准心、热量等）。
    - `_minimap: MapWidget`：固定在图传区域左下角的小地图（160×160）。

> 快速记忆：  
> - **整体布局 / 左右面板 / 顶栏 / 状态栏 → `main_window.py`**  
> - **中间图传 / HUD / 小地图 → `video_area.py`**

## 4. HUD 与图传层

### 4.1 HUD 的双轨实现（重要易踩点）

项目里有两套 HUD 实现，各有职责：

- **`rm_client/ui/hud/video_hud_overlay.py`**
  - 使用 `QPainter` 直接在图传之上绘制，是真正在屏幕上看到的 HUD。
  - 负责：
    - 左上：己方 HP、电容、自瞄状态文字。
    - 顶部中央：比赛时间、锁定时的 TARGET 距离。
    - 右上：敌方 HP、能量、发弹量。
    - 中央：准心 & 步兵斜坡辅助线。
    - 底部中央：技能 CAP 进度条。
    - 底部偏右：热量条。
    - 右下角：TIME & HP 文本。
  - 之前的大部分“位置不对 / 重叠 / 中间有紫色框”问题，都在这里修复。

- **`rm_client/ui/hud/hud_overlay.py`**
  - 是一个透明的 **Widget 容器**，挂载各种 HUD 控件：
    - `HealthBar`、`CapacitorBar`、`AimStatus`、`MatchTimer`、`Crosshair`、  
      `ChassisStatus`、`SentryInfo`、`JumpTarget`、`GimbalAngles` 等。
  - 这些是「控件版 HUD」，叠加在 `VideoArea` 之上。
  - 当前版本中有部分“双轨”：
    - 比如顶部时间由 `VideoHudOverlay` 画，`MatchTimer` 控件被设为 `setVisible(False)` 避免叠在一起。
  - 近期根据操作手反馈，已在这里修过：
    - 右侧哨兵面板贴右边缘、距边 20px。
    - 左下小地图 + 底盘状态 + 跳跃目标 + 状态机的垂直布局，避免重叠。

> 修改指引：  
> - 改 **HUD 元素在画面上的具体坐标** → 首选改 `video_hud_overlay.py`。  
> - 改 **控件样式 / 字体 / 组件逻辑** → 改 `hud/*.py`（比如 `chassis_status.py`）。  
> - 改 **控件 HUD 在容器中的布置**（贴边、间距） → 改 `hud_overlay.py._layout_components()`。

### 4.2 HUD 组件目录

`rm_client/ui/hud/` 中的主要文件：

- `health_bar.py`：血量条组件（QProgressBar + 数值）。
- `capacitor_bar.py`：超级电容条。
- `aim_status.py`：自瞄状态指示（OFFLINE / NOT LOCKED / LOCKED）。
- `match_timer.py`：大号比赛时间标签控件。
- `crosshair.py`：准心控件（步兵扌型或通用十字）。
- `chassis_status.py`：底盘状态（UNLOCKED/LOCKED/LOW/HIGH）。
- `sentry_info.py`：哨兵信息卡（目标 / 状态 / 前往）。
- `infantry/jump_target.py`：步兵跳跃目标显示。
- `infantry/gimbal_angles.py`：Pitch/Yaw 显示。

## 5. 侧栏与功能面板

### 5.1 步兵左侧面板：`InfantryLeftPanel`

文件：`rm_client/ui/control/infantry_left_panel.py`

从上到下包括：

- **AUTO-AIM STATUS**  
  靶心小图标 + 状态（LOCKED / NOT LOCKED / OFFLINE）。

- **SUPER CAPACITOR**  
  绿色进度条 + 百分比（如 88%）。

- **TOTAL ENERGY**  
  电压/能量数值（如 548V）。

- **YAW**  
  Yaw 角度（如 142.6°）。

- **MINI-MAP**  
  标题 + 内嵌 `MapWidget`（态势图）。

- **TARGET SCAN**  
  标题行带 ✕ 关闭按钮；敌人列表（ENEMY #01/#02/#03 等）；`Objective` 勾选框。

### 5.2 步兵右侧面板：`InfantryRightPanel`

文件：`rm_client/ui/control/infantry_right_panel.py`

包括四块：

- **SYSTEM STATUS**：BATTERY / CPU TEMP / AMMO / LATENCY。  
- **SCORES**：TEAM A / TEAM B 分数。  
- **MODULE HEALTH**：Gimbal / Chassis / Turret 三条进度条。  
- **CHAT LOG**：只读文本框，当前主要为空占位。  

### 5.3 英雄右侧面板：`RightInfoPanel` + PLUGINS

- 文件：`rm_client/ui/control/right_info_panel.py` + `main_window.py` 中的插件区。

- **RightInfoPanel**：
  - 连接状态
  - 比赛阶段
  - 剩余时间（秒）
  - 血量
  - “复制诊断”按钮（复制当前诊断信息到剪贴板）

- **PLUGINS 区**：
  - 在 `MainWindow.add_plugin_widget()` 中动态添加插件控件。
  - 供将来扩展用。

### 5.4 角色选择：`OperatorRoleSelector`

文件：`rm_client/ui/control/operator_role_selector.py`

- 下拉框「角色:」，选项来自 `OPERATOR_ROLE_LABELS`。
- 设置 `DataCenter.current_operator_role`。
- 驱动 `_sync_layout_to_role()` 里的布局切换（顶栏、左右面板、状态栏文案）。

## 6. 小地图与雷达

- **`rm_client/ui/radar/map_widget.py`**
  - `MapWidget`：画矩形场地和红/蓝机器人点。
  - 数据来自 `DataCenter.robot_states`（一般是 0~1 范围的归一化坐标 + 队伍字段）。
  - 同一组件被复用：
    - 左侧面板中的 MINI-MAP（步兵视图）。
    - 图传左下角的小 MINI-MAP（`video_area.py` 中单独创建）。

- **`rm_client/ui/radar/robot_status_list.py`**  
  - 如需展示机器人状态列表，可在相关面板中使用。

## 7. 样式与主题

### 7.1 新样式：`styles/colors.py`

文件：`rm_client/ui/styles/colors.py`

- 定义 `HUDColors` 等颜色常量：
  - 如 `PRIMARY / WARNING / DANGER / INFO / BG_DARK / TEXT_PRIMARY` 等。
- 后续需要统一换肤或调整主题，优先改这里。

### 7.2 旧样式（Legacy）：`styles/_legacy.py`

- 文件：`rm_client/ui/styles/_legacy.py`
- 来自原始 `styles.py`，完整保留了之前的样式常量：
  - 如 `STYLE_PANEL`、`STYLE_PANEL_TITLE`、`STYLE_BTN` 等。
- 存在目的：
  - 保证旧代码中的 `from rm_client.ui.styles import STYLE_PANEL` 等导入不报错。
- 其中一部分样式仍被面板使用，一部分已经不再使用。

### 7.3 统一导出：`styles/__init__.py`

- 同时 `import *` legacy 样式并导出 `HUDColors`。
- 常见用法：

```python
from rm_client.ui.styles import STYLE_PANEL, STYLE_PANEL_TITLE, HUDColors
```

> 清理建议（将来可以按需做）：  
> - 用 IDE 搜索某个 `STYLE_` 名是否仍有引用。  
> - 没有引用后再从 `_legacy.py` 中删除。  
> - 最终可以把样式收敛到「colors + 少量 panel 样式」。

## 9. 已知注意事项与坑

1. **HUD 的两套实现不要搞混**  
   - 修改图传上具体绘制（位置、是否重叠）→ 改 `video_hud_overlay.py`。  
   - 修改控件 HUD 的布局（比如哨兵面板贴不贴右边）→ 改 `hud_overlay.py._layout_components()`。  
   - 修改组件样式（比如底盘状态的边框）→ 改对应 `hud/*.py` 或样式文件。

2. **`styles.py` 已重构为 `styles/` 包**  
   - 老的单文件 `styles.py` 已拆为：
     - `styles/_legacy.py`（老样式）
     - `styles/colors.py`（新颜色）
     - `styles/__init__.py`（统一导出）
   - 不要再在同目录下创建新的 `styles.py`，否则会与包名冲突。

3. **买活预警逻辑在服务层，不要在 UI 里硬编码条件**  
   - 相关逻辑在 `rm_client/core/service/tactical_advisor.py`。  
   - 目前逻辑避免了「一打开就弹预警」，只有在比赛进行中且对方经济满足条件时才会提示。

4. **小地图有两处，需要区分**  
   - 左侧面板中的大一点的 MINI-MAP（步兵视图）。  
   - 图传左下角的小 MINI-MAP（固定 160×160，实时态势）。  
   - 两者共用 `MapWidget`，数据来自 `DataCenter.robot_states`。

5. **状态栏的文字更新有多处来源**  
   - 初始状态、MQTT 连接结果、步兵视图下的 SYSTEM ACTIVE 状态等，都会写状态栏。  
   - 如需统一样式或节流刷新，优先修改 `main_window.set_status_message` 和 `_sync_layout_to_role` 部分。

## 10. 后续可选的优化方向

这些不影响当前使用，但如果以后有时间，可以考虑逐步进行：

- **统一 HUD 实现路径**  
  - 当前是 QPainter HUD + Widget HUD 双轨。  
  - 长远可以选择：
    - 完全 Widget 化（HUDOverlay 为主），或  
    - 完全 QPainter 化（VideoHudOverlay 为主），减少重复。  
  - 做之前务必理清每个文件的职责，避免改错导致 UI 全挂。

- **逐步清理 legacy 样式**  
  - 参照第 7 节建议，按需删除未使用的 `STYLE_` 常量。  
  - 最终让新 UI 全面使用 `HUDColors` 等更简洁的主题配置。

- **整理 `ui/control` 目录**  
  - 给每个面板类写清楚用途与挂载位置（docstring）。  
  - 通过“查找引用”确认一些旧面板（如老的诊断面板）是否已经不再使用，再决定是否删。

## 11. 一句话路线图

> 如果要继续开发这个客户端：  
> - 改**业务逻辑 / 数据来源** → 从 `rm_client/core/` 入手。  
> - 改**总体布局 / 左右面板** → 看 `rm_client/ui/main_window.py` + `rm_client/ui/control/`。  
> - 改**HUD 布局 / 视觉** → 先看 `rm_client/ui/hud/video_hud_overlay.py`，再看 `rm_client/ui/hud/` 里的各组件及 `ui/styles/`。  
> - React 那一套只是**预览用前端**，不影响比赛客户端，但可以用来打样 UI 设计。
